# Sistema de Internacionalización (i18n)

## Setup

El proyecto usa `react-i18next`. La configuración vive en `src/lib/i18n.ts`.

Los archivos de traducción están en `src/locales/`:
- `es.json` — Español (idioma por defecto)
- `en.json` — Inglés
- `de.json` — Alemán
- `fr.json` — Francés

## Uso en componentes

```tsx
import { useTranslation } from 'react-i18next';

export default function MiPagina() {
  const { t } = useTranslation();

  return <h1>{t('pages.miPagina.title')}</h1>;
}
```

El segundo argumento de `t()` es el **fallback** — se muestra si la clave no existe:

```tsx
t('pages.aiPentesting.title', 'AI Pentesting Engine')
```

Esto significa que puedes usar keys sin añadirlas a los locales todavía, y el fallback se mostrará. El proyecto usa esta estrategia extensamente.

## Estructura de los archivos de locale

Los locales siguen esta jerarquía:

```json
{
  "app": { ... },
  "common": { "loading": "Cargando...", "cancel": "Cancelar", ... },
  "nav": { "dashboard": "Panel", "reports": "Reportes", ... },
  "pages": {
    "dashboard": { ... },
    "aiPentesting": {
      "title": "Motor de Pentesting con IA",
      "logs": { "title": "Logs de ejecución" },
      ...
    },
    "reports": {
      "title": "Reportes",
      "description": "Genera reportes de seguridad completos",
      ...
    }
  }
}
```

## Cómo añadir claves nuevas

Al añadir una sección nueva (ej: el Orquestador Autónomo), hay que añadir las claves en los **4 archivos de locale**.

Ejemplo — añadir la sección del orquestador bajo `pages.aiPentesting`:

**`es.json`:**
```json
"aiPentesting": {
  ...claves existentes...,
  "orchestrator": {
    "title": "Orquestador Autónomo",
    "targetLabel": "Target (IP o dominio)",
    "targetPlaceholder": "ej: scanme.nmap.org",
    "startButton": "Iniciar campaña",
    "pauseButton": "Pausar",
    "resumeButton": "Reanudar",
    "stopButton": "Detener",
    "newCampaignButton": "Nueva campaña",
    "viewReportButton": "Ver reporte",
    "statusLabel": "Estado",
    "iterationLabel": "Iteración",
    "targetInfoLabel": "Target",
    "noActiveCampaign": "No hay campaña activa",
    "conflict": "Ya hay una campaña en curso"
  }
}
```

**`en.json`:**
```json
"orchestrator": {
  "title": "Autonomous Orchestrator",
  "targetLabel": "Target (IP or domain)",
  ...
}
```

## Claves que ya existen y son reutilizables

Claves en `common` que aplican a la sección del orquestador:

```
common.loading     → "Cargando..."
common.cancel      → "Cancelar"
```

Claves en `pages.dashboard` reutilizables:

```
pages.dashboard.running   → "En ejecución"
pages.dashboard.paused    → "Pausado"
pages.dashboard.status    → "Estado"
```
